Adding Images

Img tags are empty elements, they have no closing tag. They must have src and alt attributes. The scr atttibute tells the browser where the image can be found. Usually this will be a relative URL pointing to an image on your site.

The alt attribute is an accurate description of the image if the image cannot be viewed. If the image is just to make the page attractive and it has no meaning, you can leave the quotes empty. Alt should ALWAYS be used.

The title attribute provides additional info about the image when the user hovers over the image.

Height and Width of Images

Height and width attributes can also be added to the img tag. Images take longer to load than the HTML code that makes up a page. Its a good idea to specify the size of the image so the browser can render the rest of the page, while leaving the right amount of space for the image to load. Image sizes are more frequently being defined in CSS.

Placement is important. Browsers show images in one of two ways.

Block elements, like h1 and p, always appear on a new line. If the img is followed by a block level element, then the block level element will sit on a new line after the image is shown, like the first example.

Inline elements, like b, em, and img, sit within a block element and DO NOT start on a new line. If the img element is inside a block element, text or other inline elements will flow around the image, like in the second and third example.

Figure and Figcaption

Images often come with captions. The figure element contains an image and caption so the two are associated.
You can have more than one image inside the figure element, as long as they all share the same caption.
Browsers sometimes indent the contents of the figure element.

The figcaption element allows captions to be added to an image. It is used within the figure element.

Before HTML 5 there was no way to associate an img element with a caption. Older browsers that don't understand HTML 5 elements ignore the new elements and display the content of them.

Image Formats

JPEG should be used when there are many different colors. It is best for all photos so you can see all the subtle differences in the colors, even for snow or sky images.

GIF or PNG should be used images have few colors of large areas of the same color. When a picture has an area that is filled with exactly the same color it's known as FLAT COLOR. Logos, illustrations, and diagrams often use flat colors.

To create an image that is partially transparent there are two formats to use.

If the transparent part of the image has straight edges and it's 100% transparent save as a GIF, with the transparency option selected.

If the transparent part has diagonal or rounded edges or you want a semi-opaque transparency or a drop shadow, save as a PNG.
