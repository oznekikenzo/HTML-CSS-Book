Chapter 6: Tables


When representing info in a table, think in terms of a grid made up of rows and columns, like a spreadsheet. This chapter covers:

  • Using the four key elements for creating tables
  • Representing complex data using tables
  • Adding captions to tables

A table represents info in a grid format. Financials, TV schedules, sports results, etc. It references data on two axes. Each block is called a TABLE CELL.

In HTML a table is written out row by row.

Basic Table Structure

15 15 30
45 60 45
60 90 90

The table element starts the table. Contents are written out row by row. A closing table element ends the table.

The TR element indicates the start of each row, TR stands for Table Row. It is followed by one or more TD elements, one for each cell in the row. The row ends with a closing TR tag.

Each cell of the table is represented by a TD tag, TD stands for table data. At the end of the cell a closing TD tag is used. Some browsers automatically draw lines around the table and/or cells. This can be controlled with CSS for this, the following examples, and all tables.

Table Headings

The TH element is used just like the TR element, but its purpose is to represent the heading for either a column or row. TH stands for Table Heading.

Even if the cell has no content, the TD or TH element should be used to represent the presence of an empty cell, otherwise the table won't render properly. The first cell the the first row shows an empty TH element. See comment in html file.

Using the TH element for headings helpsuse of screen readers, improves ability for search engines to index the pages, and give greater control of the tables appearance with CSS.

The scope attribute on TH indicates whether the heading is a row or column. Browsers usually display the content to the TH element in bold and in the middle of the cell

Spanning Columns

9am 10am 11am 12am
Monday Geography Math Art
Tuesday Gym Home Ec

Sometimes you may need the cells in a table to strech across more than one column. The colspan attribute can be used to do this on TH and TD elements. The value of this attribute equals the number of columns to be spanned.

When a cell extends across more than one column, the TD or TH cells that would have been in the place of the wider cells are not included in the code.

Three elements help distinguish between the main content of the table and the first and last rows (which can contain diffrent content). These elements help people with screen readers and also allow you to style these sections differently than the rest of the table with CSS.

The thead element is used to create row and column headings for the table.

The body of the table should sit inside the tbody element.

The tfoot element is used to create the footer of the table. It follows after the tbody element is closed.

By default browsers rarely treat these three elements any differently from other elements. Designers often use CSS styles to change their appearance, so they stand out from the body.

Some Content Management Systems (CMS) will automatically enter a thead element automatically if the first row of your table contains a th element. These are built in to the CMS's editor tools.

Part of the reason for having seperate thead and tfoot elements is if you have a table taller than your screen (or, in print longer than a page) then the browser can keep the header and footer visible whle the contents of the table scroll. It is intended to make it easier for users to see which column the data is in. This functionality is not implemented by default in any current browser.

Old Code: Width and Spacing

Withdrawn Credit Balance
January 250.00 660.50 410.50
February 135.55 895.20 1170.15

All the attributes in this table have been replaced with CSS. The width attribute on the opening table tag indicated how wide the table should be. It could be used on opening TH and TD tags to specify the width of individual cells. The value is the width of the table or cell in pixels.

Columns need to form a staight line, so you often only see the width attribute on the first row (and all subsequent rows would use that setting). The opening table tag can use the cellpadding attribute to add space inside each cell of the table and to create space between each cell. The values are given in pixels.

The border attribute was used on both table and td elements to indicate the width of the border in pixels.

The bgcolor attribute was used to indicate background colors of either the entire table or individual table cells. The value is usually a hex code.
