Chapter 4: Links


Common form of links are:

  • Links from one website to another
  • Links from one page to another on the same web site
  • Links from one part of a web page to another part on the same page
  • Links that open in a new browser
  • Links that start up your email program and address a new email to someone

Writing Links

Linking to other sites, ABSOLUTE URLs

Links are created using the a element. It has an attribute called href. The value of href is the page you want to be redirected to when it's clicked.

The link text that appears between the opening a tag and closing a tag, can be clicked to be taken to the specified page.

When you link to a diffrent website, the value of the href attribute will be the full web address, which is known as an ABSOLUTE URL. It can be the path to a specific page on the website, if not it will go to the homepage of the linked site.

Browsers show links in blue with an underline by default.

Linking to other pages on the same site, RELATIVE URLs

  • Home
  • About
  • Movies
  • Contact

When linking to other pages within the same site, you don't need to specify the domain name in the URL. Use the shorthand known as the RELATIVE URL.

If all pages of the site are in the same folder, the value of the href attribute is just the name of the file.

If you have diffrent pages of a site in diffrent folders, there is a more complex syntax to indicate where the page is in relation to the current page.

Directory Structure


Structure

The image above shows the directory structure for a fictional website called ExampleArts

On larger websites it's a good idea to organize your code by placing the pages for each different section of the site into a new folder. Folders on a website can be referred to as directories.

Top Level folder (examplearts in image) is known as the root folder. It contains all the other files and folders for a website. Each section of the site is placed into seperate folder to help organization of the files.

Relationships

The relationship between files and folders on a website is described using the same terminology as a family tree.

In the diagram above, you can see some relationships have been drawn in.

The examplearts folder is a parent ofthe movies, music, and theater folders. And the movies, music, and theater folders are children of the examplearts folder.

Homepages

The main homepage (and the homepages of each section in a child folder) is called index.html. Web servers are usually set up to return the index.html file if no file name is specified.

If you enter examplearts.com it will return examplearts.com/index.html and examplearts.com/music will return examplearts.com/music/index.html.

Relative URLs

When linking to a page on your own website, you don't need to specify the domain name. Use relative URLs which are shorthand to tell the browser where a page is in relation to the current page. They are quicker to write.

This is helpful when creating a new website or learning HTML because you can create links between pages when they are only on your computer, before you have to get a domain name and upload to the web.

If all files in your site are in one folder, simply use the file name for that page.

If the site is organized into seperate folders (or directories), tell the browser how to get from the page it is CURRENTLY on to the page you are LINKING to. Define the path.

If you link to the same page from two diffrent pages you might, therefore, need to write two different URLS. Different paths.

These links make use of the same terminology (borrowed from that of family trees) you met on the previous page which introduces directory structure.

These examples are relative to the diagram above, the Directory Structure Diagram.

To link to the SAME folder: link to music reviews from music homepage, just use the file name. Reviews

To link to a CHILD folder: link to music listings from homepage. Use the name of the child folder, followed by a forward slash, then the file name. Listings

To link to a GRANDCHILD folder: link to DVD reviews from homepage. Use the name of the child folder, followed by a forward slash, then the name of the grandchild folder, followed byanother forward slash, then the file name. Reviews

To link to a PARENT folder: link to the homepage from music reviews. Use../ to indicate the folder above the current one, then follow it with the file name. Home

To link to a GRANDPARENT folder: link to the homepage from DVD reviews. Repeat the ../ to indicate that you want to go up two folders (rather than one), hten follow it with the file name. Home

When a page is live there are some other techniques used for navagating directories. There may be the name of a child folder without the name of the file. In this case the web server will usually try to show the homepage for that section. A forward slash will return the homepage for the entire site and a forward slash followed by a file name will return that file providing it's in the root directory.

Email Links

Email Kenzo

To create a link that starts up the user's email program and addresses an email to a specified email address, use the a element. This time the value of the href attribute starts with mailto: followed by the address you want the email to be sent to.

Opening Links in a new Window

Internet Movie Database (opens in new window)

If you want a link to open in a new window, use the target attribute, after the href, on the opening tag. The value of target should be _blank.

You should avoid opening links in new windows, you want people to stay on your page, hopefully the user will return to your site after you redirected them.

A common reason for opening a new window is if it points to another website. It's considered good practice to notify that link will open in a new window.

Interlude

A brief, intervening film scene or sequence, not specifically tied to the plot, that appears within a film

Prologue

A speech, preface, or introduction, or brief scene preceding the main action or plot of a film; contrast to epilouge

Top

The id attribute can be used on any element, here it's used to identify points in the page that a link will go. The value of the id should start with a letter or underscore (no numbers or characters). On a single page no two id's should be the same.

To link to an element that uses an id attribute you use the a element, but the value of href starts with the # followed by the id.
