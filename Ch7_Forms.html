Chapter 7: Forms

The term 'form' has referred to a printed document that contain spaces for you to fill in information. HTML borrows the concept of a form to refer to different elements that allow you to collect information from visitors to your site.

HTML forms give you a set of elements to collect data from users. From a simple search box to complicated insurance applications.

In this chapter we cover:

  • How to create a form on your website
  • The different tools for collecting data
  • New HTML 5 form controls

How forms work

A user fills in a form then presses a button to submit the information to the server. The name of each form control is sent to the server along with the value the user enters or selects.
The server processes the information using a programming language, JavaScript for example. The server may also store the information on it's database.
The server creates a new page to send back to the browser based on the information received.

A form can several controls, each gathering different information. The server needs to know which piece of inputted data corresponds with which form element.
To differentiate the various pieces of inputted data, information is sent from the browser to the server using name/value pairs. You should never change the name of a form control in a page unless you know that the code on the server will understand this new value.

Form Structure

This is where the form controls will appear.


Form controls live inside a form element.It should always have the action attribute and usually a method and id attribute

Every form requires the action attribute. The value is the URL for the page on the server that will receive the information in the form when it is submitted.

Forms can be submitted using one of two methods: get or post. With the get method, the values from the form are added to the end of the URL specified in the action attribute. This method is ideal for short forms, like search boxes or when you are just retrieving data from the web server (not sending info that should be added to or deleted from a database.)

With the post method the values are sent in what are known as HTTP headers. Generally you should use the post method if the form allows users to upload a file, is very log, contains sensetive data (passwords), and adds info to, or deletes info from, a database.
If the method attribute is not used, the form data will be sent using the get method.

The value of id is used to identify the form distinctly from other elements on the page (and is often used by scripts - such as those that check you have entered information into fields that require values). We will look further into the id attribute in the next chapter.

Form Controls

Text Input

Username:

The input element is used for several controls. The value of the type attribute determines what kind of input will be created. When the type attribute is "text", it creates a single line text input.

When users enter information into a form, the server needs to know which form control each piece of data was entered into. (For example, in a login form, the server needs to know what has been entered as the username and what has been given as the password). Therefore, each form control requires a name attribute. The value of this attribute identifies the form control and is sent along with the information they enter to the server.

The size attribute should not be used on new forms. It was used in older forms to indicate the width of the input text (measured by the number of characters). The value of 3 would create a box wide enough for three characters (more characters could be entered). CSS should be used to control the width of form elements. The size attribute is mentioned because it is still out there in older code.

The maxlength attribute can be used to limit the number of character a user can enter. It's value is the number of characters that can be entered. If you were asking for a year, maxlength could have a value of four.

This is another input element. When the type attribute has a value of password it creates a text box that acts just like a single-line text input, except the characters are blocked out. They are hidden so if some nosey creep wants to see your password, they can't see the sensetive data.

The name attribute indicates the name of the password input, which is sent to the server with the password the user enters.

This input can also carry size and maxlength attributes like the single-line input.

The password is only visually hidden, this doesn't mean tha the data in the password controlis sent securely to the server. Never use these for sending sensetive data like credit card numbers. Text Area

What did you think of this gig?

The textarea element is used to create multi-line text input. This is not an empty element, it should have an opening and closing tag.

Any text between the opening and closing tags will appear in the text box when the page loads. If the user doesn't delete any text between these tags, this message will be sent to the server along with whateverthe user typed. (Some sites use JavaScript to clear this information when the user clicks in the text area.)

When creating a new form use CSS to control the height and width of textarea. Older code will have the cols and rows attributes used with this element. Cols controls how wide and rows controls high the text area should be.

Radio Button

Please selcet your favorite genre:
Rock Pop Jazz

This is another input element. When the type attribute is radio, radio buttons allow the user to pick just one of a number of options.

The name attribute is sent to the server with the value of the option the user selects. When a question provides users with answers in the form of radio buttons, the value of the name attributes should all be the same for all the radio buttons used to answer that question.

The value attribute indicates the value that is sent to the server for the selected option. The value of each of the buttons in a group should be diffrent (so the server knows which option the user selected).

The checked attribute can be used to indicate which value (if any) should be selected when the page loads. The value of this attribute is checked. Only one radio button in a group should use this attribute.

Once a radio button is selected it can't be deselected, the user can only select a different option. If you are only allowing the user one option and want them to be able to deselect it (like agreeing to terms and conditions) you should ued a checkbox instead.

This is another input element! When the type attribute is checkbox, checkbox allows users to select and deselect one or more options in answer to a question.

The name attribute is sent to the server with the value of the option(s) the user selects. When a question provides users with optoins for answers in the form of checkboxes, the value of the name attribute should be the same for all the buttons that answer that question.

The value attribute indicates the value sent to the server if this box is checked. The checked attribute indicates that this box should be checked when the page loads. If used, the value should be checked.

Drop Down List Box

The drop down list box, also known as a select box allows users to select one option from a drop down list. The select element is used to create the drop down list box. It contain two or more option elements

The name attribute indicates the name of the form control being sent to the server, along with the value selected.

The option element is used to specify the options the user can select from. The words between the opening and closing tags will be showwn to the user in the drop down box.

The option element uses the value attribute to indicate the value that is sent to the server, along with the name of the control if this option is selected.

The selected attribute can be used to indicate the option that should be selected when the page loads. This would come in to play if you, the author of the web page, had a preference of what the user should select. The value should always be selected, but as seen in the code, it's simply the text, selected after the value of option.

If this attribute is not used, the first option will be shown when the page loads. If the user doesn't select an option, the first item will be sent to the server as the value for this control

The function of a drop down box is similar to the radio buttons (only one optoin can be selected). The two key factors in choosing which to use are:

  1. If users need to see all options at a glance, radio buttons are better suited
  2. If ther is a very long list of options (such as a list of countries), drop down list boxes work better.

This type of input creates a box that looks like a text input followed by a Browse button. When a user clicks the Browse button, the presentation of the window that allows users to browse for the file they want to upload will match the user's OS. The appearance of these windows cannot be controlled.

When introducing form controls, the code was kept simple by indicating the purpose of each one in text next to it. However, each form control should have it's own label element as this makes the form accessible to vision-impaired users.

The label element can be used in two ways. It can:

  1. Wrap around both the text description and the form input (as shown on the first line of the example above).
  2. Be kept seperate from the form control and use the for attribute to indicate which form control it is a label for (as shown with the radio buttons).

The for attribute states which form control the label belongs to. Note how the radio buttons use the id attribute. The value of the id attribute uniquely identifies anelement from all other elements on the page. This will be covered in the next chapter.

The value of the for attribute matches that of the id attribute on the form control it is labeling. This technique using the for and id attributes can be used on any form control. When a label element is used with a checkbox or radio button, users can click on either the form control or the label to select. The expanded clichable are makes the form easier to use, improving UI. The position of the label is very important. If users don't know where to enter information or what information to enter, they are less likely to use the form correctly.

Grouping Form Elements

You can group related form controls together inside the fieldset element. This is helpful for longer forms.

Most browsers will show the fieldset with a line around the edge to show how they are related. The appearance of these lines can be agjusted using CSS.

The legend element can come directly after the opening fieldset tag and contains a caption which helps identify the purpose of that group of form controls.

HTML 5: Form Validation


There are forms on the web that give messages if the form control has not been filled out correctly, this is known as form validation.

Traditionally, form validation has been performed using JavaScript (which will not be covered here). HTML 5 is introducing validation and leaving the work to the browser.

Vaildation helps ensure the user enters information in a form that the server will be able to understand when th form is submitted. Validating the contents of the form before it is sent to the server helps:

  • Reduce the amount of work the server has to do.
  • Enable users to see if there are problems with the form (faster than if validation were performed on the server).

At the time of printing, only Chrome and Opera supported HTML5 validation, other are expected to follow. To support older browsers web page authors are likely to continue using JavaScript to validate forms.

An example of HTML5 form validation is the required attribute, which can be used on any form element that the user is expected to fill in. This HTML5 attribute does not need a value, but in all HTML4 all attributes must have a value. So, some people give this attribute a value of required.

HTML5: Date Input

Many forms need to gather information such as dates, email addresses, and URLs. This has traditioanlly been done using text inputs.

HTML5 introduces new form controls to standardize the way that some information is gathered. Older browsers that do not recognize these inputs will just treat them as a single line input text box.

If you are asking the user for a date, you can use an input element and give the type attribute a value of date. This wil create a date input in browsers that support the new HTML5 input types.
