Chapter 2: Text

Structural Markup: the elements that you can use to describe both headings and paragraphs. Semantic Markup: provides extra information; such as where emphasis is placed in a sentence, that something you havewritten is a quotation (and who said it), the meaning of acronyms, and so on.

Headings

This is a Main Heading

This is a Level 2 Heading

This is a Level 3 Heading

This is a Level 4 Heading

This is a Level 5 Heading
This is a Level 6 Heading

HTML has six "levels" of headings. The h1 element is used for main headings, h2 is used for subheadings, etc. They vary in size with h1 being the largest and h6 the smallest. Size, text and fonts can be adjusted using CSS.

Paragraphs

A paragraph consists of one or more sentences that form a self-contained unit of discourse. The start of a paragraph is indicated by a new line.

Text is easier to understand when it is split up into units of text. For example, a book may have chapters. Chapters can have subheadings. Under each heading there will be one or more paragraphs.

To create a paragraph, surroubd the words that make up the paragraph with an opening p tag and closes it with the closing p tag. By default browsers will show each paragraph on a new line with some space between it and any subsequent paragraphs

Bold and Italic

This is how we make a word appear bold.

Inside a product description you might see some key features in bold.

By enclosing words in the b tags we make the characters bold. It represents a section of text that would be presented in a visually diffrent way (key words of a paragraph). There is no additional implied meaning to the characters, besides to make them stand out.

This is how we make a word appear italic.

It's a potato Solanum teberosum.

Captain Cook sailed to Austrailia on the Endeavour.

By enclosing words in the i tag characters appear italic. The i element also represents a section of text that would be said in a different way from the surrounding content - such as technical terms, foreign words, thoughts, and names of ships.

Whitespace

When a browser comes across two or more spaces next to each other, it only displays one space. If it comes across a line break, it treats it as a single space as well. This is called white space collapsing. Adding whitespace makes the code easier to follow.

Paragraphs and heading automatically appear on new lines. If you wanted to add a line break in the middle of a paragraph use the tag br /.

To create a break between themes - a change of topic or new scene in a play a horizontal rulecan be added between the sections using the hr / tag.

There are a few elements that don't have any words between an opening and closing tag. They are known as EMPTY ELEMENTS. Usually only one tag, before the closing angled bracket there is a space and forward slash, as seen above.

Semantic Markup

There are elements that are not intended to affect the structure of web pages, but they do add extra information to the pages - they are known as semantic markup.

Browsers often display the contents of these elements in different ways. They should not be used to change the way the text looks; their purpose is to describe the content of your web pages more accurately. They are used so other programs like screen readers or search engines can use the extra info.

The voice of a screen reader may add emphasis to words inside the em element or a search engine might register that the website features a quote if you use the blockquote element.

Strong & Emphasis

The strong element indicates that its content has strong importance. By default browsers show this element in bold.

The em element indicates emphasis that subtly changes the meaning of a sentence. By default, this element will show contents in italic.

Quotations

Did you ever stop to think, and forget to start again?

The blockquote element is used for longer quotes that take up an entire paragraph. The p element is still used inside the blockquote. Browsers tend to indent the contents of a blockquote. Do not use this just to indent a piece of text - use CSS to achieve that effect.

As A.A. Milne said, Some people talk to animals. Not many listen. That's the problem.

The q element is used for shorter quotes that sit within a paragraph. Browsers are supposed to put quotes around the q element, IE does not - therfore many avoid using the q element.

Both elements can use the cite attribute to indicate where the quote is from. It's value should be a URL that will have more information about the source of the quotation.

Abbreviations & Acronyms

Prof Stephen Hawking is a theoretical physicist and cosmologist.

If using an abbreviation or acronym the abbr element can be used. The title attribute in the opening tag is used to specify the full term.

NASA do some crazy space stuff.

HTML 4 had a seperate acronym element. The title attribute was used to spell out the full form of the acronym. HTML 5 uses the abbr element for both.

Citation & Definitions

A Brief History of Time by Stephen Hawking has sold over ten million copies worldwide.

When referencing a piece of work (book,film,or reseach paper) the cite element can be used to indicate where the citation is from. It should not be used for a person's name, but it was allowed in HTML 4, so it's most likely to be used for that. Browsers will render the contents in italics.

A black hole is a region of space from which nothing, not even light, can escape.

The first time you explain new terminology in a document, it's known as defining the instance of it. The dfn element is used to indicate the defining instance of the new term.

Some browsers show the content in italics. Safari and Chrome do not chane the appearance.

Author Details

homer@example.org

742 Evergreen Terrace, Springfield.

The address element is used to contain the contact details for the author of the page. It can have the physical address, but doesn't have to. It can also have the phone number or email address. Browsers often display the content in italics. There is also an hCard microformat for adding address information to your markup.

Changes to Content

It was the worst best idea she ever had.

The del element shows text that has been deleted from a document with a line through the text. The ins element shows text that has been inserted to a document with an line under the text.

Laptop computer:

Was $995

Now only $375

The s element indicates something is no longer accurate or relevant, but should not be deleted. Visually the content will have a line through the center. Older versions of HTML had a u element, to underline, but that's being phased out.
